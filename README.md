# Система для регистрации, учёта, контроля заказов в ресторане
***
## Соответсвование требованиям:
### Соответствие к требованиям по устройству программы:
1. Система хранения данных будет реализована в программе-сервере. На ней 
будут храниться, и в режиме реального времени обновляться все данные по 
заказам, такие как: ресторан(название), меню(название), позиция 
меню(название, цена), заказ(номер заказа, состав заказа, статус выполнения 
заказа).
2. Результат работы для пользователя - возможность добавлять в приложение свой 
ресторан, меню, заполнять позиции меню, собирать заказы и отслеживать 
результаты их выполнения. Данный результат можно протестировать.
3. Система будет состоять из двух отдельных программ.

### Соответствие требованиям на оценку отлично:
1. Система должна работать с несколькими клиентскими сеансами, выводить 
данные в режиме реального времени. Это подразумевает многопоточность и 
сложную структуру.
2. Программа-клиент и программа-сервер будут обмениваться данными и 
запросами через сетевое взаимодействие.
3. Данный проект не предусматривает использование нейросети или машинного 
обучения. Вариант с их использованием лишь рассматривается.
***

## Функционал:
- создание страницы ресторана на платформе
- создание нескольких меню внутри ресторана
- создание позиций разных категорий внутри меню
- создание заказа
- страница позиций находящихся в производстве
- автоматическое закрытие заказа при выполнении всех входящих позиций
- отображение статистики за определенный период
***
### Roadmap:
1. Создание функционала для добавления, изменения и получения обьектов из БД (решил использовать json)
2. Добавление аунтефикации (в зависимости от уровня пользователя - администратор или работник, на сайте будет отличаться функционал)
3. Вывод функционала на сайт (клиент будет на сайте, выбрал для этого Django)
4. Создание вывода статистики
